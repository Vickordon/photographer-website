# Quick Start Guide

## 🚀 One Command Start

### Prerequisites
- Docker (20.10+)
- Docker Compose (2.0+)

### Start the Application

```bash
# Make script executable (first time only)
chmod +x start.sh

# Start with one command
./start.sh
```

That's it! The application will:
1. Build Docker images
2. Create database
3. Run migrations
4. Seed initial data
5. Start the application

### Access

- **Website:** http://localhost:4000
- **Admin Panel:** http://localhost:4000/login

## 📋 Manual Docker Start

```bash
# Build images
docker-compose build

# Start services
docker-compose up -d

# View logs
docker-compose logs -f

# Stop services
docker-compose down

# Stop and remove data
docker-compose down -v
```

---

## 🔧 Configuration

### Environment Variables

Copy `.env.example` to `.env` and configure:

```bash
cp .env.example .env
```

Required variables:
- `SECRET_KEY_BASE` - Auto-generated by start.sh
- `DATABASE_URL` - Already configured in docker-compose.yml
- `SMTP_*` - Optional, for email notifications

## 🐛 Troubleshooting

### Port 4000 already in use
```bash
lsof -ti:4000 | xargs kill -9
```

### Database connection error
```bash
docker-compose restart db
docker-compose logs db
```

### App won't start
```bash
docker-compose build --no-cache
docker-compose logs app
```

## 📊 Useful Commands

```bash
# View all containers
docker-compose ps

# View logs
docker-compose logs -f app
docker-compose logs -f db

# Access database
docker-compose exec db psql -U photographer photographer_prod

# Backup database
docker-compose exec db pg_dump -U photographer photographer_prod > backup.sql

# Restore database
cat backup.sql | docker-compose exec -T db psql -U photographer photographer_prod

# Clean up
docker-compose down -v
```
